## âœ‰ï¸ Git Message Format

This repo adheres to the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification.
Commit messages are enforced through [commitlint](https://github.com/conventional-changelog/commitlint) and a [husky](https://github.com/typicode/husky) [commit-msg hook](https://github.com/wayofdev/next-starter-tpl/blob/master/.husky/commit-msg).

This leads to more readable messages that are easy to follow when looking through the project history and also allows to generate changelogs automatically.

Check out more information about the commitlint and other linters in the [linting section](/guide/linting).

### â†’ Allowed prefixes

- **feat** â€” A new feature
- **fix** â€” A bug fix
- **perf** â€” A code change that improves performance

- **deps** â€” Changes to the package.json or package-lock.json
